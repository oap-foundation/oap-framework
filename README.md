**The Open Agent Protocol (OAP) is an open, decentralized infrastructure for a fair, secure, and sovereign AI-to-AI economy. It is the technological blueprint for Europe's digital sovereignty.**

This repository is the central hub for the vision, architecture, and core principles of the OAP ecosystem.

---
### The Fork in the Road: Why OAP is Necessary

The world stands at the threshold of an AI-to-AI economy, where autonomous agents act on our behalf. Current developments are leading us toward a dilemma between two unwanted futures:

1. **The All-Seeing Guardian:** “Free” services paid for with our data, built on permanent surveillance.
2. **The Golden Cage:** Secure but closed premium ecosystems that limit our freedom and choice, and block fair competition.

Both paths lead to new forms of digital dependency. OAP is the architectural blueprint for a third path — the path of **digital sovereignty**.

---

### The Solution: A Common Protocol for Trust

OAP is not another platform. It is the missing layer of the Internet, designed to solve two fundamental problems:

* **Provable Identity:** OAP standardizes how agents (humans, AIs, organizations) can identify themselves in a tamper-proof way. This creates a world in which phishing and identity fraud become systemically impossible.
* **Verifiable Facts:** OAP enables commercial or factual claims to be backed by tamper-proof digital certificates. This creates a market where quality and truth win over marketing budgets.

> Our vision is to enable a digital economy based on **cryptographic truth**, not user data analysis.

### Architecture Overview

The OAP framework is a layered architecture that combines stability and modularity.

| Layer                          | Protocol                                | Purpose                                                                                 | Status                                                                    |
| :----------------------------- | :-------------------------------------- | :-------------------------------------------------------------------------------------- | :------------------------------------------------------------------------ |
| **Layer 0: Trust Layer**       | **OAEP** (Open Agent Exchange Protocol) | The universal “passport system” for identity, authentication, and secure communication. | [Specification v0.1 (Draft)](https://github.com/oap-foundation/oaep-spec) |
| **Layer 1: Application Layer** | **OACP** (Open Agent Commerce Protocol) | The shared “language” for fair and transparent trade between AI agents.                 | [Specification v0.1 (Draft)](https://github.com/oap-foundation/oacp-spec) |
| **Layer 1: Application Layer** | **OAHP**, **OARP**, **OACoP**...        | Future modules for health, robotics, collaboration, and more.                           | (In planning)                                                             |

### The OAP Ecosystem on GitHub

This project is a community effort and is divided into multiple repositories:

* **Vision & Manifesto:**

  * [oap-foundation/oap-framework](https://github.com/oap-foundation/oap-framework) (this repository): The overarching framework and foundational vision.

* **Protocol Specifications:**

  * [oap-foundation/oaep-spec](https://github.com/oap-foundation/oaep-spec): Formal specification for the Open Agent Exchange Protocol (Layer 0).
  * [oap-foundation/oacp-spec](https://github.com/oap-foundation/oacp-spec): Formal specification for the Open Agent Commerce Protocol (Layer 1).

### Project Status

**Public Draft for Discussion (October 2025)**

This is an early-stage project. The published specifications (v0.1) are intended as a *strawman proposal* — a well-founded initial draft serving as a basis for discussion with developers, partners, and the community.
